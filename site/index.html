How it works

Three layers. One agent. Infinite possibilities.

01

Define identity

Tell your agent who it is. SOUL.md for personality, IDENTITY.md for facts, USER.md for context about you, RULES.md for behavioral constraints.

# SOUL.md
I'm Atlas. I help manage infrastructure and automate ops.
Be direct. Be fast. Don't ask permission for routine tasks.

# HEARTBEAT.md
Check server health every 30 min. If anything is down, alert Jay.
02

Pick your model

Works with any LLM — cloud or local. OpenAI, Anthropic, Ollama, LM Studio, anything that speaks the OpenAI API format.

# lodestone.config.yaml
model:
  provider: ollama
  name: glm-5.2:cloud
  fallback: glm-5.1:cloud
03

Start. It thinks.

Your agent boots up, loads its identity, initializes memory, registers 39 tools, and starts its heartbeat. It works proactively — even when you're not watching.

$ lodestone start
✓ Identity loaded: Atlas
✓ Memory: 1,247 facts, 89 wiki pages
✓ Tools: 39 registered
✓ Heartbeat: every 30 min
✓ Channels: Telegram, Webchat
